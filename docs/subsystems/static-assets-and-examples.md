# Subsystem: Static Assets and Examples

Static assets are served by Vite from `public/` and copied into `dist/` during
build.

## Files

| Path | Role |
| --- | --- |
| `public/viewer/index.html` | FKLD/FOLD viewer iframe source. |
| `public/examples/*.fkld` | Bundled FKLD examples. |
| `public/examples/*.fold` | Bundled FOLD examples, if present. |
| `public/examples/*.stl` | Bundled mesh examples, if present. |
| `scripts/gen-sample-stl.mjs` | Helper for generated STL samples, if used. |
| `dist/` | Generated production copy. |

## Source of Truth

Edit `public/`, not `dist/`.

`dist/` is generated by:

```sh
npm run build
```

## Load Sample Flow

`AppController.loadSample` fetches a configured URL under `public/examples/`.
In development and production, Vite serves that path relative to the app root.

Current sample constants live in:

```text
src/controller/app-controller.ts
```

## Adding an Example

1. Add the source file under `public/examples/`.
2. If it is a mesh example, ensure the extension is supported (`.obj` or ASCII
   `.stl`).
3. If it is a foldable example, ensure it has vertices, edges, and faces.
4. Update the sample constant or viewer dropdown if it should be default.
5. Run `npm run build` to refresh `dist/`.
6. Do not manually edit `dist/examples`.

## Example Quality Checklist

For FKLD/FOLD examples:

- `vertices_coords` exists.
- `edges_vertices` exists.
- `faces_vertices` exists.
- `edges_assignment` length matches `edges_vertices` when present.
- FKLD extension arrays have matching lengths.
- 3D Sim enablement works if the example should be simulatable.

For STL examples:

- ASCII STL, not binary.
- Contains valid `facet`/`vertex` records.
- Coarse enough for current pipeline tests.
- If generated, document the generator script and parameters.

## Common Failure Modes

| Failure | Cause | Fix |
| --- | --- | --- |
| Sample fetch fails | App opened via `file://` or wrong path. | Serve with `npm run dev` or fix URL. |
| Viewer loads but metadata empty | File lacks FKLD/FOLD arrays. | Validate source example. |
| Mesh conversion fails at import | Binary STL or malformed OBJ. | Re-export as ASCII STL/OBJ. |
| `dist` differs unexpectedly | Build regenerated hashed assets. | Treat `dist` as generated output. |
